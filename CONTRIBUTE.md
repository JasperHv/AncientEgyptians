# Contributing to Ancient Egyptians Game

Thank you for your interest in contributing to this project! We appreciate your help in making this project even better.

## How to Contribute

We follow a simple process for contributors to ensure smooth collaboration. Please follow these steps:

### 1. Fork the Repository
Start by forking the repository to your GitHub account:
- Click on the **Fork** button at the top right of the repository page.

### 2. Create a New Branch
Create a new branch from `main` for the feature or fix you're working on:
```bash
git checkout -b feature/YourFeature
```
Replace `YourFeature` with a descriptive name for your branch.

### 3. Make Your Changes
Make your changes in the new branch. Ensure your code follows the style guidelines below (if applicable). Be sure to write meaningful commit messages.

### 4. Commit Your Changes
Once you're satisfied with your changes, commit them with a descriptive message:
```bash
git commit -m 'Add some feature'
```

### 5. Push Your Changes
Push your changes to your fork:
```bash
git push origin feature/YourFeature
```

### 6. Open a Pull Request
Once you've pushed your branch, open a pull request (PR) on GitHub.

- Go to the repository on GitHub.
- Click the "Pull Requests" tab, then "New Pull Request".
- Select your fork and the branch you worked on, then submit the PR against the main branch of this repository.

## Code Style Guidelines
Please follow the coding style guidelines to maintain consistency in the codebase:

- **Java Code Style**: Use the default Java conventions for naming, formatting, and indentation.
- **Commits**: Write clear, concise commit messages. For example: `Fix bug in card handling`, `Add feature to shuffle deck`.

You can check the [Google Java Style Guide](https://google.github.io/styleguide/javaguide.html) for reference.

## Issues
If you plan on working on a large feature or change, it's helpful to open an issue before you begin. This ensures no one else is already working on it and provides a space to discuss your approach.

## Testing
If you add new features, be sure to include tests for them. If you fix a bug, make sure to add tests that demonstrate the bug has been resolved.

## Thank You!
We appreciate your help in improving this project! If you have any questions or need clarification, feel free to reach out via an issue or pull request.
